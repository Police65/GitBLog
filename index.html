Comandos basicos
Primer paso para usar la Git Bash

Git Bash es una terminal que ofrece una interfaz de línea de comandos similar a la de Unix en sistemas Windows. Además de trabajar con Git, también puedes utilizar Git Bash para realizar tareas en tu sistema de archivos, incluyendo la búsqueda y localización de carpetas.


Para inciar un proyecto, primero debes localizar la carpeta del proyecto, puedes usar el comando "cd"




Inicializar

El comando para incializar es 

git init⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀




una vez realizado este comando, se creara en el proyecto una carpeta llamada ".git" esta carpeta se recomienda no tocar

Dentro de la carpeta ".git" se almacena el historial completo de commits, las ramas y etiquetas, los metadatos de los archivos y otras configuraciones específicas del repositorio. Además, también contiene los objetos de Git, que son los blobs (contenidos de archivos), árboles (estructura de directorios) y commits que representan el estado y los cambios en el repositorio a lo largo del tiempo.

Es importante destacar que la carpeta ".git" y su contenido son vitales para el correcto funcionamiento de Git. Modificar o eliminar archivos dentro de esta carpeta puede provocar daños irreversibles en el repositorio y la pérdida de datos. Por lo tanto, se recomienda no manipular directamente el contenido de la carpeta ".git" a menos que se tenga un conocimiento profundo de cómo funciona Git internamente.

Añadir archivos al commit

Antes de realizar un commit, si compruebas con 

git status⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀



Git te indicara que aun no has realizado ningun commit, y te mostrara los archivos que esten dentro del commit, a su vez tambien los archivos modificados o no añadidos al commit (Por defecto los commit son en la rama en la que te encuentres)

Para añadir los archivos que desees al commit, puedes realizar el comando 

git add (archivo o directorio)⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀







Realiza tu primer commit

Antes de poder realizar un commit, debe realizarse la configuracion de la cuenta de GitHub, el propio git te da los comandos que debes utilizar para configurar tu cuenta, como se puede ver en la bash, debes enlazar el correo de la cuenta de GitHub que estes utilizando
Una vez hecho este comando, te aparecera una ventana de inicio de sesion, en la cual terminaras de vincular tu cuenta, para asi finalmente tener tu cuenta de GitHub enlazada a Git




Crea tu repositorio en GitHub

Crear un repositorio en GitHub es sencillo y te permite empezar a compartir tu código con otros desarrolladores de manera rápida y sencilla. Aquí tienes los pasos básicos para crear un repositorio en GitHub:

  • Inicia sesión en tu cuenta de GitHub o regístrate si aún no tienes una.
  • Una vez que hayas iniciado sesión, haz clic en el botón "Nuevo repositorio" en la página principal de GitHub.
  • En la página "Crear un nuevo repositorio", ingresa un nombre para tu repositorio. Elige un nombre descriptivo que refleje el propósito o contenido de tu proyecto.
  • Opcionalmente, puedes agregar una descripción para proporcionar más detalles sobre tu repositorio.
  • Decide si deseas que tu repositorio sea público o privado. Los repositorios públicos son visibles para todos, mientras que los repositorios privados solo son accesibles para ti y las personas a las que les otorgues permisos.
  • Si lo deseas, puedes inicializar tu repositorio con un archivo "README". Este archivo se utiliza comúnmente para proporcionar información general sobre tu proyecto.
  • Puedes agregar un archivo de licencia si deseas especificar los derechos y restricciones de tu código.
  • Selecciona las opciones adicionales que desees, como agregar un archivo ".gitignore" para especificar los archivos que deben ser ignorados por Git, o agregar un archivo de configuración.
  • Finalmente, haz clic en el botón "Crear repositorio" para crear tu repositorio en GitHub.




Enlazar a la GitBash

Enlazar tu nuevo repositorio a tu proyecto desde la GitBash es realmente sencillo, pues a la hora de crear el proyecto, git te dara los comandos para enlazarlo, y para realizar tu primer push. En GitHub, "push" se refiere a la acción de enviar tus cambios locales al repositorio remoto. Cuando realizas un push en GitHub, estás actualizando el repositorio remoto con los commits que has realizado en tu repositorio local.

Para enlazar el repositorio solo tienes que copiar y pegar el comando de: 

git remote add origin⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀




Clonar repositorio ya existente

Clonar un repositorio en GitHub es una habilidad fundamental para cualquier desarrollador que quiera colaborar en proyectos o trabajar en su propia copia local del código. En esta guía, te mostraré paso a paso cómo clonar un repositorio en GitHub y empezar a trabajar en él en tu entorno local.

Paso 1: Copiar la URL del repositorio

  • Abre el repositorio en GitHub que deseas clonar.
  • Haz clic en el botón verde "Code" cerca de la parte superior derecha del repositorio.
  • Selecciona la opción "HTTPS" y copia la URL del repositorio al portapapeles.

Paso 2: Clonar el repositorio en tu máquina local

  • Abre tu terminal o línea de comandos en tu sistema.
  • Navega hasta el directorio donde deseas que se guarde el repositorio clonado.
  • Ejecuta el siguiente comando para clonar el repositorio: 
git clone <URL del repositorio >⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀⠀



Reemplaza <URL del repositorio> con la URL que copiaste anteriormente. Pega la URL en tu línea de comandos y presiona Enter.

Paso 3: Verifica el repositorio clonado

  • Después de ejecutar el comando, Git descargará todos los archivos del repositorio en tu máquina local.
  • Una vez completado el proceso, podrás ver una nueva carpeta en tu directorio con el nombre del repositorio.
  • ¡Felicidades! Has clonado exitosamente el repositorio en tu máquina local.

Paso 4: Trabaja en el repositorio clonado

  • Navega al directorio del repositorio clonado utilizando la terminal o tu explorador de archivos.
  • Ahora puedes modificar los archivos, agregar nuevos commits y trabajar en el proyecto como lo harías con cualquier otro repositorio local de Git.
  • Utiliza los comandos de Git, como "git add", "git commit" y "git push", para realizar cambios y enviarlos de vuelta al repositorio remoto en GitHub.
